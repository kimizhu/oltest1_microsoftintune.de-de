# Verwalten von Microsoft Intune
Der von Microsoft gehostete Clouddienst [!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)] wird automatisch gewartet, sodass Sie nur wenige oder keine Wartungsmaßnahmen ausführen müssen. Sie sollten sich jedoch mit den folgenden Wartungsaufgaben vertraut machen, die unter Umständen für die Aufrechterhaltung Ihres Diensts erforderlich sind:

-   [Häufige Wartungsaufgaben](../Topic/Maintain_Microsoft_Intune.md#BKMK_Maintenance)

-   [Ändern der Kontakteinstellungen](../Topic/Maintain_Microsoft_Intune.md#BKMK_ManagePreferences)

-   [Verwalten von Partnerbeziehungen](../Topic/Maintain_Microsoft_Intune.md#BKMK_ManagePartners)

-   [Außerbetriebsetzen Ihres Abonnements](../Topic/Maintain_Microsoft_Intune.md#BKMK_DecommissionIntune)

## <a name="BKMK_Maintenance"></a>Häufige Wartungsaufgaben

|Aufgabe|Details|
|-----------|-----------|
|**Update des [!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)]-Diensts**|[!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)] ist ein Clouddienst. Dienstupdates werden von Microsoft für Sie verwaltet. Wenn Sie sich mit einem Verwaltungsportal im Internet verbinden, wird die Verbindung stets zur aktuellen Version hergestellt.<br /><br />Normalerweise werden durch Dienstupdates keine sichtbaren Änderungen verursacht. Wichtige Updates werden jedoch im Bereich **Schwarzes Brett** bekanntgegeben, der angezeigt wird, wenn Sie sich nach einem Dienstupdate bei der [!INCLUDE[wit_adminconsole](../Token/wit_adminconsole_md.md)] anmelden. Sie können Hinweise auch im Arbeitsbereich **Warnungen** der [!INCLUDE[wit_adminconsole](../Token/wit_adminconsole_md.md)] auf der Seite **Hinweis** ansehen.<br /><br />Wenn Updates an der Clientsoftware erfolgt sind, werden Sie möglicherweise aufgefordert, die aktuelle Clientversion herunterzuladen und bereitzustellen.<br /><br />Weitere Informationen finden Sie unter [Microsoft Intune-Dienstupdates](http://technet.microsoft.com/library/dn292747.aspx).|
|**Sichern und Wiederherstellen von [!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)]-Daten in der Cloud**|Zur Infrastruktur und zu den Daten, die im Rahmen des [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)]-Diensts für Sie verwaltet werden, gehören:<br /><br />-   Konfigurationen, die Sie an Ihrer [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)]-Instanz vorgenommen haben<br />-   Daten und Status der verwalteten Geräte<br />-   Daten, die Sie in den Cloudspeicher hochladen, wie bereitgestellte Software und Lizenzinformationen<br />-   Benutzerkonten, die Sie Ihrer Instanz von Microsoft Azure AD hinzufügen **Note:** Daten, die Sie zur Verwendung mit [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] hochladen, werden in der Cloud beibehalten. Dies ersetzt jedoch nicht die Sicherung der Quelle dieser Daten vor Ort.<br />Informationen dazu, wie Ihre Daten von Microsoft verwaltet werden, finden Sie im Whitepaper [Microsoft Intune Privacy and Data Protection Overview (Übersicht über Datenschutz und Schutz von Daten in Microsoft Intune)](http://download.microsoft.com/download/C/C/8/CC8065C4-829F-4635-B731-1D39287444C0/Windows_Intune_Privacy_and_Data_Protection_Overview.pdf).|
|**Erneutes Installieren von On-Premises Exchange Connector**|Ein Sichern oder Wiederherstellen dieses Connectors ist nicht möglich. Wenn Sie diesen Connector installieren oder erneut installieren, wird ein zuvor konfigurierter Connector von dieser neuen Instanz ersetzt.|
|**Erneutes Installieren von Service to Service Connector**|Die Konfiguration dieses Connectors wird in Ihrem Cloudspeicher beibehalten. Ein Sichern oder Wiederherstellen ist nicht erforderlich. Wenn Sie diesen Connector konfigurieren, wird ein zuvor konfigurierter Connector von dieser neuen Instanz ersetzt.|
|**Sichern von einzelnen Geräten und Computern**|In [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] wird kein Sichern oder Wiederherstellen von Daten, Einstellungen oder Anwendungen auf mobilen Geräten oder Computern verwaltet, welche mit dem Dienst verwaltet werden.|

## <a name="BKMK_ManagePreferences"></a>Ändern der Kontakteinstellungen

|Aufgabe|Details|
|-----------|-----------|
|**Ändern der Kontakteinstellungen**|[!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)]-Administratoren haben die Option, produktbezogene Mitteilungen von Microsoft zu erhalten:<br /><br />**Verwalten Ihrer eingehenden Mitteilungen**:<br /><br />1.  Klicken Sie im [Microsoft Intune-Kontenportal](https://account.manage.microsoft.com/) auf **Mein Profil**.<br />2.  Geben Sie im Bereich **Kontakteinstellungen** Ihre Kontaktdetails ein und wählen Sie die Mitteilungen aus, die Sie erhalten möchten.<br />3.  Klicken Sie auf **Speichern**.<br /><br />Sie erhalten auch dann weiterhin E-Mails zu Ihren Abrechnungs- und Dienstkonten, wenn Sie angegeben haben, dass Sie keine zusätzlichen Produktinformationen erhalten möchten.|

## <a name="BKMK_ManagePartners"></a>Verwalten von Partnerbeziehungen
Wenn Sie zurzeit nicht mit einem Partner zusammenarbeiten, können Sie auf der Website [Microsoft Pinpoint](http://go.microsoft.com/fwlink/?linkid=235605) nach einem Partner suchen. Die Verfügbarkeit von Partnern hängt von den verwendeten Diensten sowie von dem Land bzw. der Region ab, in dem/der Sie die Dienste verwenden.

|Aufgabe|Details|
|-----------|-----------|
|**Verwalten des Partners für Ihr Abonnement**|Partner sind Berater und bieten Expertenwissen zu Vertrieb, Support und Technik an, um Sie bei der Einrichtung und Wartung Ihres Abonnements zu unterstützen. Sie können jederzeit Microsoft-Partner zu Ihrem Abonnement hinzufügen und bestehende Partner bearbeiten oder entfernen.<br /><br />Um einen Partner für Ihr Abonnement hinzufügen zu können, benötigen Sie die Microsoft-Partner-ID des Partners.<br /><br />**Verwalten eines Partners für ein Abonnement**:<br /><br /><ol><li>Klicken Sie im [Microsoft Intune-Kontenportal](https://account.manage.microsoft.com/) auf **Verwalten**.</li><li>Klicken Sie auf [!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)].</li><li>Klicken Sie auf der Seite **Abonnementdetails** im rechten Bereich unter **Partnerinformationen** auf:<br /><br /><ul><li>**Hinzufügen**, um einen Partner zu Ihrem Abonnement hinzuzufügen.</li><li>**Bearbeiten**, um einen Partner zu ändern oder zu entfernen.</li></ul></li><li>Aktualisieren der Daten für die **Microsoft-Partner-ID**:<br /><br /><ol><li>Geben Sie die Partner-ID des Partners ein, mit dem Sie zusammenarbeiten möchten, um diesen hinzuzufügen oder zu ändern.</li><li>Löschen Sie die Partner-ID, um einen Partner zu entfernen</li></ol></li><li>Klicken Sie auf **OK**, um die Änderung zu speichern.</li></ol>|
|**Verwalten eines delegierten Administrators**|Wenn Sie [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] abonnieren, erhalten Sie Administratorberechtigungen und können anderen Benutzern in Ihrem Unternehmen Administratorberechtigungen erteilen. Wenn Sie jedoch wünschen, dass Personen außerhalb Ihres Unternehmens den Dienst verwalten, können Sie diese Rolle an einen Microsoft-Partner delegieren. Wenn Sie einen Partner zur Übernahme dieser Rolle autorisieren, wird der Partner als *delegierter Administrator* bezeichnet.<br /><br />**Hinzufügen eines delegierten Administrators**: Dieser Vorgang muss von Ihrem Microsoft-Partner initiiert werden.<br /><br />1.  Der Partner sendet Ihnen eine E-Mail, in der Sie gefragt werden, ob Sie dem Partner Berechtigungen für die Funktion als delegierter Administrator erteilen möchten.<br />2.  Lesen Sie die Bestimmungen des Partners in der E-Mail.<br />3.  Wenn Sie dem Vertrag zustimmen möchten, klicken Sie auf den Link in der Nachricht, um eine Autorisierungsseite zu öffnen.<br /><br />**Anzeigen der delegierten Administratoren**:<br /><br />1.  Klicken Sie im [Microsoft Intune-Kontenportal](https://account.manage.microsoft.com/) unter **Support** auf **Übersicht**.<br />2.  Klicken Sie auf der Seite **Supportübersicht** unter **Delegierte Administratoren** auf **Delegierte Administratoren verwalten**.<br /><br />**Entfernen eines delegierten Administrators**:<br /><br />Wenn Sie einen delegierten Administrator entfernen, entfernen Sie die Berechtigungen des Partners für den Zugriff auf Ihren Dienst und für das Ändern des Diensts. Sie können den delegierten Partner jederzeit entfernen.<br /><br />1.  Klicken Sie im [Microsoft Intune-Kontenportal](https://account.manage.microsoft.com/) unter **Support** auf **Übersicht**.<br />2.  Klicken Sie auf der Seite **Supportübersicht** unter **Delegierte Administratoren** auf **Delegierte Administratoren verwalten**.<br />3.  Wählen Sie auf der Seite **Delegierte Administratoren** den Partner aus, den Sie entfernen möchten, und klicken Sie auf **Delegierten Administrator entfernen**.|

## <a name="BKMK_DecommissionIntune"></a>Außerbetriebsetzen Ihres Abonnements
Verwenden Sie die folgenden Informationen, um Daten von Geräten und aus Ihrer Infrastruktur zu entfernen, wenn Sie [!INCLUDE[wit_firstref](../Token/wit_firstref_md.md)] nicht mehr verwenden.

|Aufgabe|Details|
|-----------|-----------|
|**Kündigen Ihres Abonnements**|Um Ihr Abonnement zu kündigen, müssen Sie sich [an den telefonischen Support für Microsoft Intune wenden](http://technet.microsoft.com/library/jj839713.aspx).<br /><br />Aus Datenschutzgründen werden Sie gebeten, die folgenden Informationen zu Ihrem Konto anzugeben:<br /><br />-   Arbeits- oder Schulkonto für Anmeldung beim Clouddienst<br />-   Vor- und Nachnamen des primären Kontakts für Ihre Organisation<br />-   Der Name Ihrer Organisation, der für das Konto angezeigt wird<br />-   Straße und Hausnummer Ihrer Organisation<br /><br />Bevor Sie Ihr Abonnement kündigen:<br /><br />-   **Entfernen von Domänen aus Ihrem Clouddienst**: Wenn Sie für die Verwendung mit [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] eigene Domänennamen für Ihre Organisation hinzugefügt haben, müssen Sie [eine Domäne entfernen](http://msdn.microsoft.com/library/azure/jj151817.aspx#bkmk_remove).<br />-   **Sichern Ihrer Daten**: Die Ihrem Onlinedienst zugeordneten Daten werden 90 Tage nach der Kündigung Ihres Abonnements gelöscht.<br /><br />Sie können Ihr Abonnement innerhalb von 90 Tagen nach Kündigung reaktivieren, indem Sie sich telefonisch an den Support wenden.|
|**Entfernen von Daten von Geräten und Computern**|Wenn Sie beabsichtigen, den Dienst nicht mehr zum Verwalten von Geräten zu verwenden, können Sie die mit [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] verwalteten Anwendungen sowie die [!INCLUDE[wit_iwportal_1](../Token/wit_iwportal_1_md.md)]-App entfernen und die [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)]-Clientsoftware auf den Computern deinstallieren.<br /><br />Normalerweise bleiben die Anwendungen und Konfigurationen für [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] bestehen, nachdem Sie die Verwendung von [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] einstellen. Deshalb gilt Folgendes:<br /><br />-   Planen Sie, die integrierten Optionen zum Zurücksetzen von Geräten zu verwenden, um vertrauliche Informationen zu entfernen.<br />-   Planen Sie die Deinstallation von Anwendungen, bevor Sie die aktive Verwaltung beenden. Der Vorgang zum Deinstallieren von Anwendungen ist von der jeweiligen Anwendung abhängig.<br />-   Beachten Sie, dass zuvor festgelegte oder erzwungene Einstellungen auf dem Gerät beibehalten werden. Wenn die Einstellungen nicht mehr erzwungen werden, können Sie vom Gerätebesitzer geändert werden.<br /><br />Informationen zum Abkoppeln von Geräten von [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] finden Sie in den folgenden Themen:<br /><br />-   **Mobile Geräte**: Siehe unter [Schützen Ihrer Daten mit Remotezurücksetzen, Remotesperre und Kennungszurücksetzung in Microsoft Intune](../Topic/Help_protect_your_data_with_remote_wipe,_remote_lock,_or_passcode_reset_using_Microsoft_Intune.md).<br />-   **Computer:** Weitere Informationen finden Sie unter [Allgemeine Aufgaben zur Verwaltung von Windows-PCs mit dem Microsoft Intune-Computerclient](../Topic/Common_Windows_PC_management_tasks_with_the_Microsoft_Intune_computer_client.md) im Abschnitt „So koppeln Sie einen Computer ab“.|
|**Entfernen der Infrastruktur**|[!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)]verfügt über keine gemeinsame Infrastruktur oder Software, die Sie von Ihren Servern deinstallieren. Es folgen jedoch einige Beispiele für optionale Tools, die Sie möglicherweise mit [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] verwendet haben und deinstallieren können, wenn sie in Ihrer Umgebung nicht mehr benötigt werden:<br /><br /><ul><li>On-Premises Exchange Connector</li><li>Tools zur Active Directory-Synchronisierung einschließlich:<br /><br /><ul><li>Tool zur Verzeichnissynchronisierung</li><li>Microsoft Active Directory-Verbunddienste 2.0 (AD FS)</li><li>AD FS 2.0 Proxy</li><li>Anderer Identitätsanbieter</li></ul></li></ul>Stellen Sie vor dem Deinstallieren eines Tools oder einer Lösung sicher, dass sie nicht mehr verwendet werden. Beispielsweise könnten Sie eine Lösung wie das Active Directory-Synchronisierungstool zum Synchronisieren Ihrer lokalen Benutzer mit Azure AD verwenden. Wenn Sie Abonnements anderer Clouddienste wie Office 365 weiterhin nutzen, müssen Sie das Tool möglicherweise beibehalten, weil es von diesen Diensten benötigt wird.|
|**Entfernen von Daten aus der Cloud**|Wenn Sie [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] nicht mehr abonnieren möchten, können Sie das [!INCLUDE[wit_icp_2](../Token/wit_icp_2_md.md)] und die [!INCLUDE[wit_adminconsole](../Token/wit_adminconsole_md.md)] verwenden, um in der Cloud gespeicherte Daten aktiv zu entfernen. Zu diesen Daten gehören Informationen zu Benutzern und Geräten, zu hochgeladener und bereitgestellter Software sowie zu benutzerdefinierten Konfigurationen wie Gruppen und Berichten.<br /><br />Wenn Sie Ihre Daten nicht aus der Cloud entfernen, bevor Ihr Abonnement endet, werden diese für eine Toleranzperiode von 90 Tagen beibehalten. In dieser Zeit können Sie weiterhin auf Ihre Daten in [!INCLUDE[wit_nextref](../Token/wit_nextref_md.md)] zugreifen und diese verwalten. Sie können jedoch keine Geräte mehr mit dem Dienst verwalten.<br /><br />Nach Ablauf der Toleranzperiode werden alle Instanzen Ihrer Daten automatisch von Microsoft gelöscht und sind dann nicht mehr verfügbar. Weitere Informationen finden Sie unter „Data disposition“ („Datenverfügbarkeit“) im Whitepaper [Microsoft Intune Privacy and Data Protection Overview (Übersicht über Datenschutz und Schutz von Daten in Windows Intune)](http://download.microsoft.com/download/C/C/8/CC8065C4-829F-4635-B731-1D39287444C0/Windows_Intune_Privacy_and_Data_Protection_Overview.pdf).|
